Vision

A roguelike dungeon crawler built for couch co-op. Every run starts clean — no meta-progression, no permanent unlocks. Players pick from a roster of four distinct classes, each with a fixed weapon and different stat profile, then descend through 10 floors of dungeon rooms. Challenges vary — combat arenas, mazes, puzzles, boss encounters. Items found along the way define the run. A single dungeon environment throughout; beat floor 10 to win. Pixel art, top-down, Zelda-paced.

Room Types

Room types are assigned by a per-room weighted roll after the dungeon graph is generated. Boss is the only fixed count (exactly 1 per floor, random placement). All others are probabilistic — average counts assume ~13 rooms per floor.

Combat Phase 1 · 58%
The default (~7 per floor). Clear all enemies to unlock the exit doors. Semi-open layout with random tetris-block cover obstacles — some rooms are fully open, most have light cover. Cover is randomly generated per room.
Boss Phase 4 · fixed 1
Exactly 1 per floor, randomly placed in any room except the spawn room — independent of the stairs. Floors 1–9: mini-boss (same structure, weaker stats). Floor 10: final boss. Open arena layout. Telegraph: passageway tiles leading into this room are colored gold.
Maze Phase 4 · 17%
~2 per floor. Recursive backtracking wall layout, enemies scattered throughout. Navigation is the challenge — no special clear condition beyond reaching the exit.
Chest Phase 4
No combat. Guaranteed item drop on entry — upgrade or equipment. Spawn chance TBD.
Puzzle Phase 4 · tabled
Tabled for a later phase — needs proper design space. Interactive objects trigger environmental changes (e.g. lever removes fire blocking a door). Currently filled by Maze rooms in the floor composition.
Trap Tile Phase 4 · ~1%/room
Not a dedicated room type — a tile that can spawn inside any eligible room (excludes shop, shrine, boss). ~1% chance per eligible room, giving ~10% cumulative odds per floor. Warps all players forward N floors. Players can see it and opt not to step on it.
Shop Phase 5 · 17%
~2 per floor. Spend HP to buy from a shared item pool — one pool for the whole team, so players decide together who gets what. No gold system needed.
Healing Shrine Phase 5 · 8%
~1 per floor. No enemies. Restore partial HP on entry — a deliberate breath room. Each player heals independently on entry.
Wave Phase 5
Enemies spawn in sequential waves. Doors don't open until all waves are cleared. Upgrade drops after the final wave. Spawn chance TBD.
Bounty Phase 5
One named champion enemy with a guaranteed rare item. Spawn chance TBD.
Darkness Phase 5
Severely limited visibility radius around each player. Enemies harder to spot; combat becomes more tense and unpredictable.

FLOOR 0 Foundation Done ▶
The game exists and runs.
  • ✓ Phaser 3 client + Colyseus authoritative server, connected and running
  • ✓ Knight — playable character with full animations and sword overlay
  • ✓ Two enemy types (green slime, red slime) with AI state machine
  • ✓ Co-op 1–4 players (P key to add), each with its own server connection
  • ✓ Melee combat with tunable swing angles
  • ✓ Tile-based map, wall collision, tile effects (fire / slime)
FLOOR 1 Rooms & Floors Done ▶
Milestone: Enter a room, clear it, walk through a door, and descend a floor.
  • ✓ Dungeon generator: 5×4 grid of 21×16-tile rooms, random walk with 60% neighbor chance, seeded RNG, barriers between rooms
  • ✓ Room type assignment: weighted roll per room (58% Combat / 17% Maze / 17% Shop / 8% Shrine / 1 fixed Boss)
  • ✓ Combat room carve function: semi-open layout with randomly placed tetris-block cover (25% chance fully open)
  • ✓ Boss room placement: random non-spawn room; gold breathing passageway tiles as in-world telegraph
  • ✓ Enemy spawning per room: type-aware (skips boss/shop/shrine), count scaled by floor & player count
  • ✓ Door lock/unlock: lock on entry if enemies present; unlock on clear; empty rooms never lock
  • ✓ Stairs: stepping on stairs tile descends to next floor; difficulty scales
  • ✓ Room transition: camera pan on room entry (current behavior accepted)
Art needed: None for MVP — doors, stairs, and cover blocks can be colored tiles. Gold passageway tiles for boss are a tile color change.
FLOOR 2 Characters & Projectiles Done ▶
Milestone: All four classes are playable with distinct-feeling weapons.
  • ✓ Character + weapon pickers before floor 1 loads; per-class stats wired in (class × skin as separate axes)
  • ✓ ProjectileState schema + GameState.projectiles MapSchema; server ticks all active projectiles (kinematic, no matter-js body)
  • ✓ Rogue: fast thrown dagger/star, 1 hit, despawns on wall hit
  • ✓ Ranger: traveling arrow, 1 hit, despawns on wall hit
  • ✓ Mage: staves fire a per-staff elemental bolt (rangedStyle: "cast"); the AOE nova is built and reserved as a Mage ability
  • ✓ Client renders projectile entities from AMMO_REGISTRY
Landed early beyond scope: the full Super Overhead Adventure 2 pack (12 humanoid skins, 29 enemies, 8 bosses); a unified combat resolver + single Spell type (windUp→strike→active→recover) shared by players, enemies, and bosses; five elemental bolts with cast + nova client FX.
FLOOR 3 Upgrade System Done ▶
Milestone: Picking up an item changes how the rest of the run plays. Decision: first pass is stat boosts only — active abilities come later (cheap to add on the new Spell system).
  • ✓ Shrine rooms hold a reward pedestal; bosses drop one where they die
  • ✓ Reward picker: room pauses, 1-of-3, first player to claim it takes it (shops stay unpaused — a shop decision is reversible, this one isn't)
  • ✓ First upgrade set: 8 stat boosts (+max HP, +speed, +damage, +armor, +lifesteal), flat and percent variants
  • ✓ Floor-gated upgrade pool (minFloor) + weapon-modifier magnitudes that scale with depth
  • ✓ Upgrades are per-player, held on the Player, cleared on run end
  • ✓ Per-instance weapons: each wielded weapon is its own object with its own modifiers; reward pedestals roll them
  • ✓ Attack pipeline: each stage (player → weapon instance → ammo → target) contributes to the Attack; Caster.scaleAttack is the single fold point
  • — Later: active abilities (dash, shield, projectile modifiers) as Spells
  • — Later: damage types + per-enemy vulnerabilities (slashing/fire/blunt) — the attack pipeline is built to accept this, but the taxonomy, per-enemy resistances, and weapon balance are their own pass
Art needed: First pass can use colored shapes + text labels. Proper upgrade icons are nice-to-have.
FLOOR 4 Room Variety Done ▶
Milestone: Two rooms in a row that feel completely different. Met — a floor now mixes combat, maze, shop, shrine and chest rooms, with traps as a cross-cutting hazard. Called done on chests + traps; puzzle rooms deferred rather than rushed.
  • ✓ Chest rooms: 7% spawn weight, no combat. One chest at room center; interact for a guaranteed weapon — no choice, no cost, contents never previewed. Brown rolls 1 modifier, the 15%-rare gold rolls 2
  • ✓ Trap tiles: 1% roll per eligible room (never boss/shop/shrine/chest, never the start room); warps the whole party 1–3 floors forward. Rendered as a violet tile in plain sight, kept off doorways and room centers, so it's always avoidable
  • — Puzzle rooms: deferred out of Floor 4 — lever/switch objects trigger environmental changes (fire, wall tiles, doors). Needs a dedicated design pass before any code: what a switch targets, how state syncs, and whether a puzzle can ever softlock a room. Picked up in Floor 5, where the class movement abilities give them something to be built around and the "small, optional, multiple solutions" scope makes the softlock question moot
Already landed: the OO boss framework (aim-lock, movement behaviors) + three full movesets — Turtle Dragon (rock projectiles, Tremor Slam, Shell Spin), Wyvern (flying, diving swoop), Tengu Mask (three abilities). Shop & Healing Shrine rooms also exist. Chest art is imported (6-frame open animation, brown + gold). Art still needed: the trap tile is a flat violet placeholder, consistent with the other five flat-color tiles (only FLOOR has real art) — the whole tileset wants one art pass together.
FLOOR 4.5 Playtest Response Up Next ▶
Milestone: Two people can play a full run together without a bug ending the session. Everything here comes from the first external playtest — full notes and the decision log (D1–D18) in docs/playtest-2026-07-20.md.
1. Session-enders — these end a co-op run outright.
  • — Barrier locks the second player out — if two players don't cross a doorway in the same tick, one is ejected back. Fix: one-way barriers — latecomers walk in, nobody walks out. Committed on entry, but the room starts without the whole party
  • — P (add local player) spawns invisible, inside a wall — late-joiner spawn isn't validated against walkable tiles and the sprite never attaches
  • — Menu rebuild — a pause menu you can resume from. Escape closes the topmost overlay; quitting a run is explicit and confirmed. (Today Escape always quits: "you can pause, but you can never unpause")
2. Cheap feel wins
  • — Interaction prompts ("press F") on chests, shop pedestals, offer pedestals, stairs — the tester couldn't tell a chest from scenery
  • — Room-entry grace period so a door-adjacent enemy can't free-hit you
  • — Slower enemies, more of them — encounters should read as positioning, not reflex
  • — Enemy line-of-sight: no aggro through walls (also stops bosses chasing into the entry corridor)
  • — Universal death feedback — several enemies currently just stop moving
  • — Telegraph incoming waves before they spawn
  • — Lock camera zoom at default; remove the zoom-out option (it's a workaround for samey textures — fix the textures instead)
  • — Make the stairs read as stairs — the tester didn't register there were multiple floors
  • — Bug sweep: projectiles pass through closed barriers · boss stun-lockable against a wall · walk-through chests · oversized crossbow/hammer icons · axes don't mirror · ghost "Wave 1/3" banner · oversized boulder hitboxes · knockback-immune enemies
  • — Move respawn behind the debug menu — the design is permadeath
  • — Rebind weapon switch off Q/E
3. Co-op actually works
  • — Per-player boss rewards — each player picks 1 of 3, picks mutually exclusive across the party. (Today it's first-come: "Hey! You took my reward!")
  • — Party stairs prompt — "Player X wants to descend. Join them to continue"; the floor advances only once every player is on the stairs. Descending without clearing stays intentional — trading loot for speed is a real choice
  • — Enemy HP scales with party size — ×1.0 / 1.5 / 2.0 / 2.5 for 1–4 players, fixed at run start. Scale the enemies, never nerf the player for bringing a friend
  • — Minimap — called out emphatically, and load-bearing once players split up to find the exit
Design note: splitting the party to clear a floor faster was a play pattern we hadn't planned for and now want to support. It creates the tension the whole phase is built around — stick together for power, split up for speed. One-way barriers and the minimap are its prerequisites.
FLOOR 5 Systems: Economy, Classes, Lobby Up Next ▶
Milestone: A run has an economy, classes have an identity, and strangers can find each other. Built in dependency order — currency, then lobby, then class powers, then puzzles.
Currency — replaces the HP-cost shop.
  • — Enemies drop gold into one shared party purse. Shared deliberately: a contested resource makes players negotiate, and that's better co-op than four isolated wallets. (Rewards are personal, money is communal)
  • — Shop tiers at 50g / 100g / 150g
  • — Budgeted, not per-enemy-priced. State the intent as constants — TARGET_SPEND_PER_PLAYER_PER_FLOOR = 100, TARGET_CLEAR_FRACTION = 0.7 — and derive the floor's gold budget (≈143g × players). SpawnDirector divides it across what it spawned, weighted by a goldWeight getter on each Enemy subclass. Enemy and room counts can then change freely without breaking the economy. No id→gold table
  • — Tier costs stay fixed and the per-floor budget stays flat across floors — what scales with depth is the shop's quality. Keeps "one item per floor" true on floor 9 and keeps prices learnable
  • — Gold sinks: the shop, plus shrines now cost gold. Boss and chest offers stay free picks; gold may appear as one of the choices on an offer pedestal
  • — Rushing a floor short-changes you — that shortfall is the cost of speed, and it's what makes descending early an interesting choice rather than a free one
Classes get an identity
  • — Weapon categories restricted per class — Knight: hammers + maces (6) · Mage: staves (6) · Ranger: bows + crossbows (3) · Rogue: daggers + thrown (6). Swords, axes, spears and rapiers (31) stay open to everyone, so ~74% of the pool is shared and loot stays varied
  • — Restriction is one-way: weapons are restricted to classes, classes aren't restricted to weapons. A staff is Mage-only; the Mage may still swing a broadsword. Reversible if classes read too weakly
  • — Type-based, never string-based — each class declares its usable weapon categories as a compiler-checked list on its CharacterConfig. Adding a category without assigning it is a build error, not a silent "nobody can use this"
  • — Loot rolls filtered by party composition — no staves drop without a Mage present. A filter on the roll, not a post-hoc reject, so the odds stay correct
  • — Movement ability per class, on its own cooldown: Blink (through bars/grates, over gaps) · Jump (over gaps) · Charge (through breakable walls) · Dash (distance). Never through locked barriers
  • — Class special power — Knight first: a regenerating shield, so the melee class can hold the front line without paying HP for it. Build it as the pattern; Rogue/Ranger/Mage powers get designed once we've seen it work
Menus & matchmaking
  • — Choose Solo or Shared at game start
  • — Shared shows a browsable room list, or create your own. Rooms are public-listed or private-by-code, player's choice
  • — Lobby-only joins — no dropping into a run in progress
Puzzle rooms — unblocked, because the movement abilities are what they're built around.
  • — Scope that finally makes them safe: small, optional, multiple solutions. Because nothing behind a puzzle is required, no puzzle can strand a party — which was the exact softlock worry that deferred them out of Floor 4
  • — Obstacles that answer to a specific movement ability: a chest behind bars (Blink), across a gap (Jump/Blink), behind a crumbly wall (Charge)
  • — Pressure plates — a second player stands on it, or shove a boulder onto it. Torches to light (or extinguish) to reveal a chest. A pot on a plate whose removal springs a trap or a secret
  • — Material solutions: burn wooden barriers with magic, shatter stone with a hammer, blow up crumbly walls
  • — Four-Swords-style co-op: boulders that need two players, plates that need multiple bodies
Open, tracked in docs/playtest-2026-07-20.md §7: what each class starts with · whether rolls should favor the party's exclusive categories rather than merely permit them · whether ClassPower is a new OO family or the Knight's shield is just an always-on Upgrade the class is born with · how much of the lobby is real matchmaking vs. UI over Colyseus room listing.
FLOOR 5.5 Content Pass Later ▶
Milestone: Two consecutive runs feel different enough to want a third.
  • ✓ More enemy types — 29 imported (goos, bats, floaters, critters, directional, flyers), spawn-pooled in REGULAR_ENEMIES
  • ✓ More bosses — 3 movesets built; 8 boss sprites available in BOSSES
  • ✓ Healing Shrine + Shop room variants
  • ✓ Wave rooms — 3 escalating waves on one RoomChallenge substrate
  • ✓ Darkness rooms — client-only vision overlay, no server state
  • ✓ Timed-clear rooms — beat the clock for a bonus pedestal; missing it is not a failure
  • — Defend-the-objective room (the rest of the old "Bounty" slot) — needs a destructible entity
  • — More upgrades: weapon enhancements, passive abilities
  • — Per-weapon special attacks beyond the current stab/smash — AOE blast, buckshot spread, single heavy shot. Possibly "different weapons cast different spells" rather than spells being a Mage-only pool. The nova is already built and sitting unused
  • — Enemy pool shifts by floor range within the dungeon theme
Mostly landed early: the enemy roster and multiple bosses came in with the SOA2 pack. Remaining work here is the last room variants, floor-scaled enemy pools, and deeper upgrades — not art.
FLOOR 6 Release Prep Later ▶
Milestone: A complete stranger can sit down and play without help.
  • — Win screen (cleared floor 10) and death screen with run summary
  • — Sound effects + ambient music — the tester noticed the silence within three minutes. Biggest single gap between this and something that feels like a game
  • — Controller support — gamepad input exists but is untested. M/kb is the default only because it's easiest to test, and it doesn't feel great
  • — Weapon balance pass — all 42 weapons are visually final and none are balanced against each other
  • — Texture variety pass so rooms are distinguishable at the locked zoom (see Floor 4.5)
  • — Difficulty balancing pass across all 10 floors
  • — Itch.io build; Steam prep if scope warrants
Endgame & replay — direction, not yet scheduled.
  • — A defined ending with meta-unlocks (Hades / Isaac), not endless scaling. Endless was explicitly flagged as a failure mode — it goes unplayable rather than climactic. Unlock weapons, classes, items and bosses across runs; track per-class completion
  • — Multiple themed dungeons, each unlocked by clearing the last — gated hard by art budget, since a new dungeon means a new enemy set
  • — Plan of record: ship v1 with one level set. Only justify a second set if people like the first
Art needed: Title / menu art. Minimal new sprites if content is solid by Floor 5.

Open Questions
Resolved — July 2026
Stat boosts only for the first upgrade set, or active abilities too?
Decided: stat boosts first. +max HP / +speed / +damage / +lifesteal, per-player, floor-scaled — fast to build and balance, gets the run-defining loop working end-to-end. Active abilities (dash, shield, projectile modifiers) come as a later pass, and are cheap to add now that everything casts through the unified Spell system.
Resolved — July 2026
In co-op, who picks a reward — each player, or the first to reach it?
Decided: first player to claim it. One pedestal, one winner. Much simpler than per-player choice lists, and it creates a real conversation between teammates about who needs the pick. Revisit if it turns out to feel mean rather than tense.
Open — direction noted
How far does weapon modification go?
Scope north-star: Souls-like breadth. Many weapon types sharing artwork, each dealing different damage types, each individually modifiable — rolled stats on spawn, upgrades applied to a specific weapon, not just to the player. This is the target the per-instance weapon work and the attack pipeline are built toward; it is deliberately not all being built at once. Damage types are the first deferred slice.
